5 motivos pelos quais o controle de versão é importante:
    1- Evitar perda de arquivos, permite recuperar versões anteriores;
    2- Permitir várias pessoas trabalharem no mesmo projeto sem sobrescrever o trabalho da outra;
    3- Registrar quem alterou, o que alterou e quando alterou;
    4- corrigir erros rapidamente, se algo der problema, pode voltar para uma versão funcional;
    5- Permitir criar ramificações e testar funcionalidades novas sem afetar o projeto principal.

Exemplos onde o controle de versão seria útil:
    Quando várias pessoas estão programando diferentes partes em um sistema ao mesmo tempo, quando acontece algum erro e você pode 
    voltar facilmente a versão anterior, pra testar uma nova funcionalidade sem comprometer o projeto principal.

Diferença entre um sistema centralizado e distribuido:
    No sistema centralizado, existe um servidor central onde são armazenadas todas as versões, os usuários precisam está logados
    a esse servidor para poder baixar ou enviar mudanças.
    No sistema distribuido, cada desenvolvedor tem uma cópia completa do repositório na sua máquina, isso permite trabalhar mesmo
    sem internet, e o envio das alterações é enviada depois para o servidor.
