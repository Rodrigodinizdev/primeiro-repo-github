1. git log
Mostra o histórico completo de commits com detalhes completos.
Inclui: autor, data, hash completo, mensagem do commit e possível diffs.
É útil quando queremos analisar profundamente o histórico.

2. git log --oneline
Mostra o histórico de forma resumida:
apenas o hash reduzido
mensagem do commit
É ideal para ter uma visão rápida e compacta dos commits, especialmente quando o repositório tem muito histórico.

3. git log --oneline --graph
Mostra o mesmo resumo do --oneline, porém com um gráfico ASCII representando a estrutura dos branches.
É muito útil para visualizar merges, bifurcações e o fluxo do desenvolvimento.

4. git log -n 2
Exibe apenas os dois commits mais recentes.
Pode-se usar outros números também (-n 5, -n 10).
Perfeito quando você quer verificar apenas o que foi feito recentemente.

5. git log --author="Seu Nome"
Filtra o log para mostrar apenas commits feitos por um autor específico.
Útil para pesquisar seu próprio histórico ou o de alguém da equipe.
